Radare2 (R2) debugging assembly is fun.

Before I stumbled over r2 when I had to debug assembly it was a task to be dreaded with out the help of something like IDA pro. Unfortunately I can’t always have things I want (expensive software licenses in this case), so I went out in search of an open source alternative. I was pleasantly surprised by the epic software that I found. Radare and it’s suite. Radare is a suite of tools designed to perform disassembly, assembly, analysis, cryptanalysis, patching, and a whole smorgasboard of other awesome operations on binaries. The whole suite of tools is crowned by it’s tool R2. R2 is a tool that brings all of the other unix-style tools in the suite together.

Intro to Ragg2

Jul 30th, 2012

Ragg2 A glorified attack string generator

Ragg2 is a part of the radare suite of tools (which can be found at https://github.com/radare/radare2.git). Ragg is described by the suite as a tool to execute programs in exotic environments and as a front in for r_egg which is a tool for compiling code that is relocatable and injectable into other binaries (shellcode). This post will describe some common tasks that are preformed with ragg (directed at solving the first few exploit exercises).

For installation of radare refer to the install document included in the radare repo.

To demonstrate ragg we generate the strings needed to solve the first few protostar exploit exercises from exploit-exercises.com.

Exploit Exercise 0:

To solve this you simply have to change the have to change the int (that will be allocated right next to buffer (refer to my post on basic stack overflows for more information)) to anything other then what it starts as. This can be simply performed by creating a buffer of 68 characters and using it as input to your program. While this is a trivial task, being lazy is a virtue so we are going to use ragg2 to do it for use here is how

This command consists of two flags. The -p flag tells ragg2 that you would like to generate padding the A68 parameter tells ragg2 to generate 68 A’s (the a can be capital or lowercase here but it will matter later). -r tells ragg to output as raw bytes not hexpairs.

Exploit Exercise 1:

This is the same as the last but requires you to put a specific number into the int, normally there are a few things that make this slightly less trivial chief among them is taking care of endianness. ragg will do this for you here is how.

The new parameter -d allows you to patch in a double word into the generated string. The parameter works as pos:hex-double-word.

The next few exploit exercises are the exact same as the previous ones. The only feature left that I am going to cover is the primary function of ragg2 which is to generate shell code.
